Recently I had the pleasure of taking the JavaScript Masterclass with noted JavaScript guru Thomas Fuchs and his brilliant accomplice Amy Hoy. The class was taught over 2 days, each day consisting of a four hour session. Eight hours might not sound like a lot, but there was a wealth of information included.

What they covered

For someone who has been doing JavaScript development for a while, this class had quite a bit of refresh material. Don't let that put you off. Amy and Thomas do a great job of shedding new light on the core of JavaScript, and it doesn't hurt to have an expert or two answering questions via video.

The JavaScript language

Thomas and Amy do a good job of explaining JavaScript's "everything is an object" nature, it's object model, and it's nearly functional design.

It wasn't anything terribly new for me, but I did get rocked on a couple things, and it was really great to have an expert walk you through and confirm your assumptions. A classic example of augmenting JavaScript was taught in the class, and it exhibited much of what makes JavaScript so awesome:

{% highlight js linenos %} //a real exercise solution I had to cook up to a class assignment: Function.prototype.cached = function() { var self = this, cache = {}; return function(args) { if(args in cache) return cache[args]; return cache[args] = self(args); }; }; Math.sin = Math.sin.cached(); {% endhighlight %}

This tiny example exhibits a lot of what is taught: JavaScript and the prototype, closures, and the functional powers of JavaScript.

In addition to the core features of the language, Thomas and Amy will drive meta programming into your face - and hard at that. You will learn how to use curry powers and call functions dynamically and use the powers of JavaScript reflection.

{% highlight js linenos %} for(name in people.Names) people['hello' + name] = function(name, msg) { return msg + ' ' + name; }.curry(name); //now use the dynamic meta power! people.helloBrian('You are cool'); // You are cool Brian; {% endhighlight %}

JavaScript Infrastructure

So many of us (myself included) have had a tendency to look at JavaScript as an exception to "real programming". This part of the course is definitely what I needed to hear and see in action. Scripts and their structure tend to be jumbled into disparate massive minified scripts, or no separation at all exists, and you end up with the same massive file for development and production.

Thomas and Amy show you how to drop this nasty habit and organize your scripts into a helpful folder structure like you would with any other program. The focus of the class was on Sprockets, a nice little tool in ruby with a command line script that can be used.

Given this structure:

You can have a script set up like so:

{% highlight js linenos %} (function(){ //= require "models/mymodel" //= require "models/mymodel2" })(); {% endhighlight %}

You can then run the sprocketize command like so:

{% highlight bash %} $ sprocketize -I . main.js > main.prod.js {% endhighlight %}

Voila - your scripts are included into the closure above. No more hassle of dealing with one jumbo script. Thomas and Amy show how this is an invaluable tool, and your JavaScript can and should be treated like any other application code.

One thing to note: The class seems to assume use of Sprockets version 1. Sprockets 1 is a bit outdated at this point, but it gives the desired functionality above. For some reason, Sprockets 2 dropped the ability to have require directives inside of a closure - you are now forced to include directives at the top of a script. This makes it less useful in my opinion, and after wrestling with Sprockets 2 for a hot second, I decided to drop it for Sprockets 1. So when you install that bad brotha, make sure you save yourself a headache:

{% highlight bash %} $ gem install --remote sprockets -v 1.0.2 {% endhighlight %}

Unit testing

I'm no stranger to unit testing, but I'll confess it is absent from my JavaScript more often than it should be. Thomas gives a real solid introduction to unit testing in JavaScript, and you will see things you should be used to in your day to day code adventures:

{% highlight js linenos %} //a style you might pick up from the class testSomeUnnamedFactoryFunction:function() { this.assertUndefined($()); this.assertNull(document.getElementById('doesNotExist')); //more asserts.... } {% endhighlight %}

They offered up expert advice, and if you are a beginner you won't be after this class, and if you are in the advanced camp it will definitely confirm your assumptions and even drop a few new goodies your way. It can't hurt to have on your resume that you studied at the feet of the masters either.
